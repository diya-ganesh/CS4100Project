# ChessBot

This project builds a modular chess engine in Python. The system is organized into separate components for board representation, search strategies, and evaluation functions, allowing different approaches to be swapped in without changing the core engine. The board is implemented using bitboards with precomputed attack tables along with Zobrist hashing to track repetitions. This makes the board representation efficient, while letting us directly manage the game state. Rather than relying on a chess library, we can better understand and optimize key operations like move generation and hashing. For decision-making, the primary algorithm is iterative deepening alpha-beta search with move ordering and quiescence search. Additionally, we include a greedy best-first search and a beam search for comparison. Our first evaluator is the material evaluator that counts how many pieces on the board with respective piece values in order to figure out whether white or black is ahead and by how much. Our second evaluator is handmade, which combines material evaluation with positional advantage tables and other positional heuristics like king safety and board control. Furthermore, it adapts strategy based on how close the game is to finishing. 

Because search and evaluation are split, we could swap algorithms and heuristics independently and compare them in the same round-robin setup. Results were recorded by printing win-loss-draw records, termination reasons, and ply counts. Key design decisions include prioritizing modularity in order to have dynamic comparisons between different algorithmic approaches instead of optimizing a single approach. The primary tools used were Python 3 and pygame for the GUI, with the chess library being used early on before transitioning into the finalized board representation.
